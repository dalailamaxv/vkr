# Безопасность

## Хранение секретов

- Все API-ключи (`OPENAI_API_KEY`, `NVIDIA_API_KEY`, `DID_API_KEY`) хранятся **только** в файле `backend-full-site/.env`.
- Файл `.env` **не должен попадать в репозиторий** — он добавлен в `.gitignore`.
- Перед публикацией репозитория всегда выполняйте проверку:

  ```bash
  git ls-files | grep -E "\.env$"
  # Должно быть пусто. Если файл показался — удалите его из отслеживания:
  git rm --cached backend-full-site/.env
  git commit -m "Remove leaked .env from tracking"
  ```

## Что делать при компрометации ключа

Если `.env` случайно попал в коммит (даже один раз), считайте все ключи скомпрометированными — git-история сохраняет всё, и удалить файл из коммита **не отменяет** его утечку.

Немедленно отзовите и перевыпустите ключи:

- **OpenAI:** https://platform.openai.com/api-keys — удалить старый ключ, создать новый.
- **D-ID:** https://studio.d-id.com/account-settings — Revoke API Key, затем Generate New.
- **NVIDIA NIM:** https://build.nvidia.com → Profile → API Keys — Invalidate, затем Create new key.

После ротации обновите локальный `.env` новыми значениями.

## Если ключ всё-таки попал в git-историю

Чтобы вычистить файл из всей истории (а не только из последнего коммита), используйте `git filter-repo` (предпочтительно) или `git filter-branch`:

```bash
# Установить git-filter-repo
pip install git-filter-repo

# Удалить файл из всей истории
git filter-repo --path backend-full-site/.env --invert-paths

# Force-push (внимание: переписывает историю)
git push origin --force --all
```

После этого **обязательно** отзовите старые ключи — даже если файл удалён из истории, его могли проиндексировать клонировщики и автоматические сканеры утечек (GitHub Secret Scanning, TruffleHog и т. п.).

## Дополнительные меры на продуктивном развёртывании

- На эндпоинте `/api/ask` включить rate limiting (`slowapi`, например 60 запросов/мин на IP).
- Для встраиваемого виджета задать `Content-Security-Policy` с явным перечнем источников скриптов, стилей и медиаконтента.
- Все обращения к внешним API — по HTTPS с проверкой сертификатов (это поведение по умолчанию у `httpx`, `requests` и `aiohttp`).
- В продуктивной среде использовать секрет-менеджеры (Docker Secrets, HashiCorp Vault, GitHub Actions Secrets) вместо файла `.env`.
- Не логировать содержимое `OPENAI_API_KEY`, `DID_API_KEY`, `NVIDIA_API_KEY` ни в каких сообщениях/исключениях.

## Защита персональных данных пользователей

Прототип сознательно не обрабатывает биометрические данные пользователя:

- Анализ эмоций реализован rule-based способом по тексту вопроса и ответа (не по фото, голосу или поведенческим признакам).
- История диалога ограничена тремя последними обменами и передаётся клиентом в каждом запросе (stateless-обработка).
- В журнале сервера сохраняются только обезличенные тексты вопросов и идентификаторы источников; персональные данные пользователей не записываются.
