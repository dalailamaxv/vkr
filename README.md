# vkr — виртуальный помощник с видео-аватаром

Программный прототип виртуального помощника на базе RAG-архитектуры (Retrieval-Augmented Generation) с синтезом речи и видео-аватара. Индексирует произвольный публичный сайт, отвечает на вопросы по его содержимому и сопровождает ответ озвучиванием с мимикой аватара.

В составе:

- **`backend-full-site/`** — серверная часть на FastAPI: индексация сайта, RAG-поиск, генерация ответа, синтез речи через Edge TTS и видео-аватара через D-ID.
- **`browser-extension-full-site/`** — браузерное расширение на стандарте Manifest V3, добавляющее перемещаемый чат-виджет с видео-аватаром на любой сайт.

---

## Возможности

- **Индексация** любого сайта по карте сайта (`sitemap.xml`), `robots.txt` или вручную заданному списку URL.
- **Поиск ответа** по корпусу страниц через RAG.
- **Обязательная ссылка на источник** — каждый ответ сопровождается ссылкой на конкретную страницу сайта.
- **Озвучивание** ответа через Microsoft Edge TTS (русский голос `ru-RU-DariyaNeural`, бесплатно, без API-ключа).
- **Видео-аватар** через D-ID Talks API с согласованной мимикой (опционально).
- **Анализ эмоциональной окраски** ответа (rule-based, без обработки биометрических данных).
- **Стриминговый режим** ответа — эндпоинт `/api/ask-stream`.
- **Голосовой ввод** через Web Speech Recognition API.
- **Пайплайнирование длинных ответов** — параллельная генерация фрагментов TTS+видео.

## Технологический стек

| Слой | Технология |
|------|------------|
| Серверный фреймворк | FastAPI 0.109 + Uvicorn |
| Векторное хранилище | ChromaDB 0.4+ (локально, он-прем) |
| Языковая модель | GPT-4o-mini (OpenAI API) |
| Модель эмбеддингов | `llama-3.2-nv-embedqa-1b-v2` (NVIDIA NIM) |
| Синтез речи | Microsoft Edge TTS (`edge-tts` 7.x) |
| Видео-аватар | D-ID Talks / Clips API |
| Парсинг страниц | BeautifulSoup 4 + lxml |
| HTTP-клиент | aiohttp + httpx + requests |
| Браузерное расширение | Chrome Manifest V3 |

## Quick Start

### 1. Требования

- **Python 3.11+** ([скачать](https://www.python.org/downloads/))
- **Git**
- Аккаунты с API-ключами:
  - [OpenAI](https://platform.openai.com/api-keys) — обязательно
  - [NVIDIA NIM](https://build.nvidia.com) — обязательно (бесплатный тариф)
  - [D-ID Studio](https://studio.d-id.com) — опционально, для видео-аватара (бесплатный кредит $5 при регистрации)

> Если ключ D-ID не задан, ассистент работает в режиме «текст + речь» без видео.

### 2. Установка серверной части

```bash
# 1. Клонировать репозиторий
git clone https://github.com/dalailamaxv/vkr.git
cd vkr/backend-full-site

# 2. Создать и активировать виртуальное окружение
python -m venv venv
# Linux / macOS:
source venv/bin/activate
# Windows (PowerShell):
.\venv\Scripts\Activate.ps1

# 3. Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt

# 4. Скопировать пример конфигурации и заполнить ключи
cp .env.example .env
# Откройте .env в редакторе и подставьте свои значения:
#   OPENAI_API_KEY, NVIDIA_API_KEY, DID_API_KEY (опц.), DID_PRESENTER_ID (опц.)
```

### 3. Запуск сервера

```bash
# Из директории backend-full-site/
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

После запуска:

- Health-check: http://localhost:8000/health
- Документация OpenAPI (Swagger): http://localhost:8000/docs
- Альтернативная документация (ReDoc): http://localhost:8000/redoc

### 4. Индексация сайта

```bash
# Через curl
curl -X POST http://localhost:8000/api/index-website \
  -H "Content-Type: application/json" \
  -d '{"base_url": "https://example.com"}'

# Проверка статуса индексации
curl http://localhost:8000/api/index-status/https%3A%2F%2Fexample.com
```

Индексация занимает 10–25 минут в зависимости от объёма сайта.

**Альтернатива — пошаговая индексация:**

```bash
# Только эмбеддинги по уже скачанным страницам
python embed_only.py https://example.com

# Индексация с обходом скрытых страниц (отредактируйте BASE_URL и MANUAL_URLS в файле)
python index_with_hidden_pages.py

# Очистка прогресса при перезапуске
python cleanup_progress.py https://example.com
```

### 5. Первый запрос

```bash
# Обычный (синхронный) ответ
curl -X POST http://localhost:8000/api/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "Расскажи, что есть на сайте?", "base_url": "https://example.com"}'

# Стриминговый ответ
curl -N -X POST http://localhost:8000/api/ask-stream \
  -H "Content-Type: application/json" \
  -d '{"question": "Какие есть разделы?", "base_url": "https://example.com"}'
```

Ответ содержит:
- `answer` — текст ответа;
- `sources` — список ссылок на страницы-источники;
- `audio_url` — путь к MP3-озвучке;
- `video_job_id` — ID фоновой задачи генерации видео (опрашивается через `/api/video-status/{job_id}`).

### 6. Подготовка приветственного видео (опционально)

```bash
# Сгенерировать короткое приветственное видео для расширения
python generate_greeting_video.py
# Результат скопируется в ../browser-extension-full-site/assets/intro/greeting.mp4
```

### 7. Установка браузерного расширения

```text
1. Откройте chrome://extensions/ в Chromium-браузере
   (Google Chrome, Microsoft Edge, Яндекс.Браузер).
2. Включите «Режим разработчика» в правом верхнем углу.
3. Нажмите «Загрузить распакованное расширение».
4. Выберите папку browser-extension-full-site/ из репозитория.
5. Иконка расширения появится в панели браузера. Откройте любой сайт —
   виджет помощника появится в правом нижнем углу.
```

## REST API

| Метод | Эндпоинт | Назначение |
|-------|----------|------------|
| `GET`  | `/health` | Health-check, проверка готовности сервера |
| `POST` | `/api/index-website` | Запуск индексации сайта по `base_url` |
| `GET`  | `/api/index-status/{base_url}` | Проверка состояния индексации |
| `POST` | `/api/ask` | Обработка вопроса, формирование ответа, запуск задачи генерации видео |
| `POST` | `/api/ask-stream` | То же, но в стриминговом режиме (Server-Sent Events) |
| `GET`  | `/api/video-status/{job_id}` | Опрос статуса фоновой задачи генерации видео |
| `GET`  | `/api/video-proxy/{job_id}` | Проксирование MP4 от D-ID через бэкенд |
| `POST` | `/api/tts` | Получение MP3-озвучки фрагмента текста |

Полная интерактивная документация доступна по адресу `/docs` после запуска сервера.

## Безопасность

- **API-ключи хранятся только в `.env`** (добавлен в `.gitignore`). Перед публикацией репозитория **всегда** проверяйте, что `.env` не попал в коммиты.
- В продуктивном развёртывании рекомендуется использовать секрет-менеджеры (Docker Secrets, HashiCorp Vault).
- Все обращения к внешним API выполняются по HTTPS с проверкой сертификатов.
- Прототип **не обрабатывает биометрические данные** (анализ эмоций — rule-based по тексту), что соответствует требованиям российского законодательства о персональных данных.
- Для встраиваемого виджета на продуктивном сайте рекомендуется задать политику `Content-Security-Policy` с явным перечнем источников.
- На эндпоинте `/api/ask` рекомендуется включить rate limiting (`slowapi`, 60 запросов/мин/IP) перед публичным развёртыванием.

Подробнее — см. [`SECURITY.md`](SECURITY.md).

## Переход на он-прем (импортозамещение)

Архитектура допускает замену внешних сервисов на локальные аналоги:

| Внешний сервис | Локальная замена | Требования |
|----------------|------------------|------------|
| GPT-4o-mini (OpenAI) | YandexGPT Lite / GigaChat | Облачный API оператора (РФ) |
| NVIDIA NIM embeddings | `sentence-transformers` (multilingual-e5-base) | 2–4 ГБ RAM, CPU |
| Edge TTS (Microsoft) | Yandex SpeechKit | Облачный API оператора (РФ) |
| D-ID Talks API | SadTalker / MuseTalk | GPU ≥ 8 ГБ VRAM, latency 2–5 сек |
